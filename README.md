## 📝 О проекте

**Filmorate** — это бэкенд-приложение для настоящих ценителей кино. Проект представляет собой REST API для социальной сети, где пользователи могут делиться своими впечатлениями о фильмах, находить друзей со схожими вкусами и открывать для себя новые картины с помощью умной системы рекомендаций.

Проект создан в рамках углублённого изучения Java-разработки с фокусом на:
*   Проектирование архитектуры веб-приложений.
*   Работу с реляционными базами данных через JDBC.
*   Реализацию сложной бизнес-логики (рекомендательная система).
*   Написание модульных и интеграционных тестов.

### ✨ Ключевые возможности

*   **Управление контентом:** Полноценные CRUD-операции для фильмов и пользователей.
*   **Каталоги:** Предопределённые справочники жанров (6 видов) и возрастных рейтингов MPA (G, PG, PG-13, R, NC-17).
*   **Социальная сеть:** Система дружбы с подтверждением (статусы PENDING/CONFIRMED).
*   **Взаимодействие:** Пользователи могут ставить и удалять лайки фильмам.
*   **Рейтинги:** Формирование топа самых популярных фильмов на основе лайков.
*   **Интеллект:** Персонализированные рекомендации фильмов, основанные на пересечении лайков пользователей (алгоритм "те, кому понравилось это, также понравилось то").

## 🧠 Архитектура и ключевые решения

Проект построен по классической многослойной архитектуре:
1.  **Controller Layer:** REST-контроллеры, обрабатывающие HTTP-запросы.
2.  **Service Layer:** Содержит бизнес-логику (управление друзьями, лайками, расчёт рекомендаций).
3.  **Storage Layer:** DAO-репозитории, отвечающие за прямые запросы к базе данных через `JdbcTemplate`.

**Ключевая особенность — реализация рекомендательного сервиса (`RecommendationService`).**
Алгоритм работы рекомендаций:
1.  Находит всех пользователей, которые поставили лайки тем же фильмам, что и целевой пользователь.
2.  Среди них выбирает "самого похожего" — того, у кого больше всего общих лайков с целевым пользователем.
3.  Из списка фильмов, которые понравились "самому похожему" пользователю, исключаются те, что уже оценил целевой пользователь.
4.  Оставшиеся фильмы возвращаются в качестве рекомендаций.

## 🚀 Инструкция по развёртыванию

### Системные требования
*   **Java Development Kit (JDK):** 21 или выше.
*   **Maven:** 3.9 или выше (или использование встроенного Maven-wrapper).
*   **Git** (для клонирования репозитория).

### Шаги для запуска

1.  **Клонируйте репозиторий:**
    ```bash
    git clone https://github.com/твой-username/твой-репозиторий.git
    cd filmorate
    ```
    *(Замени ссылку на актуальную)*

2.  **Соберите проект с помощью Maven:**
    ```bash
    ./mvnw clean install  # для Linux/macOS
    mvnw.cmd clean install # для Windows
    ```

3.  **Запустите приложение:**
    ```bash
    ./mvnw spring-boot:run
    ```
    Или запустите класс `FilmorateApplication.java` из вашей IDE.

4.  **Приложение запустится на:** `http://localhost:8080`

### Взаимодействие с приложением
*   Все эндпоинты доступны через HTTP-запросы.
*   Для тестирования API можно использовать **Postman**, **Insomnia** или просто перейти по адресу консоли H2.
*   **Консоль базы данных H2** доступна по адресу: `http://localhost:8080/h2-console`
    *   **JDBC URL:** `jdbc:h2:file:./db/filmorate` (или `jdbc:h2:mem:testdb` для тестов)
    *   **User:** `sa`
    *   **Password:** `password`

## 📊 Схема базы данных

![Filmorate DB Schema](filmorate_schema.png)

Схема базы данных включает таблицы для пользователей (`users`), фильмов (`films`), жанров (`genres`), рейтингов (`ratings`), а также промежуточные таблицы для связей:
- `film_genres` — связь фильмов и жанров (многие-ко-многим),
- `likes` — лайки фильмов пользователями,
- `friendships` — дружба между пользователями.

### Примеры SQL-запросов

### Получить все фильмы
```sql
SELECT * FROM films;

SELECT f.film_id, f.name, COUNT(l.user_id) AS likes_count
FROM films f
LEFT JOIN likes l ON f.film_id = l.film_id
GROUP BY f.film_id
ORDER BY likes_count DESC
LIMIT N;


SELECT u.*
FROM friendships fr
JOIN users u ON fr.friend_id = u.user_id
WHERE fr.user_id = :id AND fr.status = 'CONFIRMED';


SELECT u.*
FROM friendships fr1
JOIN friendships fr2 ON fr1.friend_id = fr2.friend_id
JOIN users u ON fr1.friend_id = u.user_id
